## Описание разработанного приложения

В рамках лабораторной работы была реализована серверная часть веб-приложения библиотеки.

Разработанное приложение предоставляет REST API для управления библиотечной системой. С помощью API можно выполнять операции над следующими сущностями:

- книги
- читатели
- читальные залы
- выдача книг

Система реализует стандартную архитектуру REST и поддерживает следующие операции:

- получение списка объектов  
- получение отдельного объекта по идентификатору  
- создание новых записей  
- обновление существующих данных  
- удаление записей  

---

## Модель базы данных

Для реализации системы библиотеки были разработаны следующие модели.

### Book (Книга)

Модель хранит информацию о книгах библиотеки.

Поля:

- title — название книги  
- authors — автор книги  
- publisher — издательство  
- publication_year — год публикации  
- section — раздел библиотеки  
- cipher — библиотечный шифр  

---

### Reader (Читатель)

Модель хранит информацию о читателях библиотеки.

Поля:

- имя читателя  
- номер читательского билета  

---

### ReadingRoom (Читальный зал)

Модель содержит информацию о читальных залах библиотеки.

Поля:

- название зала  
- вместимость  

---

### Borrowing (Выдача книги)

Модель связывает читателя и книгу и хранит информацию о выдаче книги.

Поля:

- reader — читатель  
- book — книга  
- borrow_date — дата выдачи  
- return_date — дата возврата  

---

## Реализация API

API реализовано с использованием Django REST Framework.

Для каждой модели были созданы сериализаторы и ViewSet-классы, которые обеспечивают работу с данными через REST API.

Каждая сущность поддерживает стандартные REST операции:

- GET — получение данных  
- POST — создание объекта  
- PUT / PATCH — обновление данных  
- DELETE — удаление объекта  

Благодаря использованию ViewSet Django REST Framework автоматически создаёт маршруты для выполнения этих операций.

## Документация API (Swagger)

Для просмотра и тестирования API используется Swagger.  
Swagger автоматически генерирует документацию для всех endpoint-ов.

Документация доступна по адресу:

```
http://127.0.0.1:8000/swagger/
```

Swagger позволяет:

- просматривать все доступные endpoint-ы  
- видеть структуру запросов и ответов  
- выполнять API-запросы прямо из браузера

## Аутентификация пользователей

В системе реализована аутентификация пользователей с использованием библиотеки **Djoser**.

Основные операции:

- регистрация пользователя  
- получение токена авторизации  
- выход из системы  

Основные endpoint-ы:

```
POST /auth/users/
POST /auth/token/login/
POST /auth/token/logout/
```

После авторизации пользователь получает токен, который используется для выполнения защищённых запросов к API.
